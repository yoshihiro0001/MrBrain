# Design Taste Test

作成日: 2026-07-02

## 目的

Design Systemを確定する前に、現在のDesign System候補がユーザーの好みとズレていないか確認する。

これはDesign System本体ではない。
採用前の好み確認シートである。

特に確認すること:
- Playfair Displayのような明朝体、筆記体、エレガント寄りフォントが本当に合っているか
- 細い線、繊細すぎるグラデーションが苦手ではないか
- ロゴっぽいベクターアイコン、太めでシンプルな図形、見やすいUIが合っているか
- アニメーションは派手ではなく、心地よく、意味があるものが合っているか

## 1. 好き / 苦手 チェック表

| 項目 | 好き候補 | 苦手候補 | 判定 |
|---|---|---|---|
| 全体印象 | ミニマル、整理、見やすい、直感的 | 装飾過多、情報過多、読みにくい | 未確認 |
| 高級感 | 静かで上質、余白がある | きらびやか、過度にエレガント | 未確認 |
| 色 | 黒、白、透明度、少ないアクセント | 色数が多い、派手なグラデーション | 未確認 |
| 線 | 太めで見やすい、意味のある区切り | 細すぎる線、繊細すぎる境界 | 未確認 |
| アイコン | シンプル、太め、ロゴ感、見れば分かる | 細かすぎる、意味が弱い、装飾的 | 未確認 |
| フォント | 読みやすい、現代的、数字が強い | 明朝体すぎる、筆記体、読みにくい装飾 | 未確認 |
| アニメーション | 意味がある、心地よい、短い | 派手、長い、目的がない | 未確認 |
| UI | 主役情報が大きい、操作が分かる | 文字が小さい、階層が弱い | 未確認 |

## 2. フォント候補

### 現在候補

| 用途 | 既存候補 | レビュー |
|---|---|---|
| 基本文字 | Inter | 採用候補。読みやすく、幅広く使いやすい |
| 数字 / 金額 / コード | JetBrains Mono | 採用候補。数字やデータに強い |
| 見出し / 装飾 | Playfair Display | 保留。高級感はあるが、エレガント寄りすぎる可能性がある |

### 確認したいこと

Playfair Displayのようなセリフ体は、HOTEL JOYや会社運営OSの世界観に合う可能性はある。

ただし、ユーザーの好みが「ロゴっぽい、太め、シンプル、見やすい」に寄っているなら、明朝体・筆記体・細いエレガント表現はズレる可能性がある。

### 代替候補

| 用途 | 候補 | 方向性 |
|---|---|---|
| 見出し | Inter / Geist / SF Pro系 | ミニマルで実用的 |
| ロゴ風見出し | 太めのSans | シンプルで強い |
| 数字 | JetBrains Mono / Tabular nums | 金額、時間、部屋番号に強い |
| 装飾フォント | Playfair Display | 使うなら限定的 |

## 3. アイコン・イラストの方向性

### 採用候補

- ロゴっぽいベクターアイコン
- 太めでシンプルな図形
- 意味が一瞬で分かるアイコン
- A4やLINEでも潰れない形
- 状態が分かるシンプルな記号

例:

| 用途 | 方向性 |
|---|---|
| 注意 | 太めの警告アイコン |
| 完了 | はっきりしたチェック |
| 撮影 | カメラアイコン |
| 触らない | 手 / 禁止マーク |
| 部屋 | シンプルな部屋番号カード |
| 料金 | 円マーク + 数字 |

### 保留

- 細い線だけの繊細なアイコン
- 意味より雰囲気優先のイラスト
- 写真風の装飾イラスト
- 複雑なSVG装飾

## 4. モーションの方向性

### 採用候補

- 短いフェード
- 軽いスライド
- 状態変化が分かるアニメーション
- 成功時の短いチェック表示
- 画面遷移を助けるMotion

### 保留 / 禁止候補

- 派手なバウンド
- 意味のないループアニメーション
- 長い演出
- 操作を待たせるMotion
- 目立つだけの装飾Motion

判断基準:

```md
動きがあることで理解が早くなるか。
動きがあることで行動しやすくなるか。
```

Yesなら採用候補。
Noなら不要。

## 5. LINE / A4 / Web / Appでの見え方

| Artifact | 合いそうな方向性 | 注意点 |
|---|---|---|
| LINE | 短い文、絵文字または記号、チェック項目、安心文 | 装飾しすぎない。長文にしない |
| A4 | アイコン中心、チェック式、余白多め、禁止事項を明確に | 黒背景やグラス表現は印刷では重い可能性 |
| Web | ダーク、ガラス、カード、主役数字、滑らかなMotion | 情報密度を上げすぎない |
| App | スマホファースト、指で押しやすい、承認が簡単 | 文字を小さくしすぎない |
| Dashboard | 主役数字、状態、未確認、次Action | 装飾より判断速度 |
| PDF | 白背景寄り、印刷可読性、余白 | グラス表現は控えめ |
| Mail | 読みやすさ、短い段落、結論先 | 高級感より誤解防止 |
| 通知 | 1行で何をするか分かる | 情報を詰めすぎない |

## 6. 採用候補 / 保留 / 禁止候補

### 採用候補

- Inter系の読みやすいSans
- JetBrains Mono系の数字表現
- 黒背景 + 白文字 + 透明度
- エメラルド、赤、黄色の状態色
- 太めで分かりやすいアイコン
- 主役情報を大きくする
- 余白とグループ化
- 短く意味のあるMotion
- A4ではアイコン + チェック + 余白

### 保留

- Playfair Display
- 明朝体 / セリフ体
- グラスモーフィズムの全Artifact適用
- 繊細なグラデーション
- 細い線だけのUI
- 写真付き説明書

### 禁止候補

- 筆記体中心のデザイン
- 文字が小さいUI
- 色数が多いデザイン
- 装飾のためのアニメーション
- 情報を詰め込みすぎたA4
- 意味が分からない抽象アイコン
- 細すぎて見えない線

## 7. 次に作るべき小さなデザインサンプル

Design Systemを確定する前に、次の4サンプルを作る。

### Sample 1: LINE Mission

目的:
Human Agentがスマホで読んで動けるか確認する。

内容:
- タイトル
- 今日やること
- チェック3つ
- 安心文
- 禁止事項
- 返送先

### Sample 2: A4 Mission Sheet

目的:
印刷して使えるか確認する。

内容:
- アイコン中心
- チェック式
- 余白多め
- 黒背景ではなく印刷向き
- 禁止事項を目立たせる

### Sample 3: Dashboard Card

目的:
主役情報が見やすいか確認する。

内容:
- 未確認件数
- 次Action
- 状態色
- 大きい数字
- 右上補足情報

### Sample 4: Mobile Approval

目的:
スマホで承認しやすいか確認する。

内容:
- 承認
- 修正
- 保留
- 却下
- 理由の短い表示

## Taste Test結論

現時点では、Design Systemを確定しない。

まず確認すべきズレ:

1. Playfair Displayが本当に合うか。
2. エレガント寄りより、太めでロゴっぽいシンプルさの方が合うか。
3. 細い線や繊細なグラデーションが苦手ではないか。
4. グラス表現を全Artifactに広げるべきか、Web / App限定にすべきか。
5. A4 / PDFは黒背景より白背景・印刷向けにすべきか。
