# 名前と意識の接続OS

## 位置づけ
これは、樫原義大OSにおける名前、ブランド、言葉、意識の接続原則である。

単なるネーミング術ではない。

名前は、機能を説明するものではなく、人の意識を接続する入口である。

## このファイルが必要な理由
ユーザーは、言葉を探しているだけではない。

言葉が人の意識をどこへ運ぶかを設計し始めている。

これは、エネルギー循環OS、Human Communication OS、投資家OS、HOTEL JOY、AI Agent、将来のアプリ名に共通する最上位の設計思想である。

## 中核原則
```md
名前は機能を説明するものではない。
人の意識を接続する入口である。
```

人は、仕組みそのものよりも、その先に得られる状態に惹かれる。

例:
- 筋トレではなく、健康
- 勉強ではなく、できるようになること
- 接続ではなく、循環
- 宿泊施設ではなく、HOTEL JOY

## 人は未来を買う
人は、機能ではなく未来を買う。

機能名は内部の構造を説明する。

ブランド名は、使った後の世界を想像させる。

だから、ブランド名と内部構造の名前は違ってよい。

むしろ、違う方が強い。

## レイヤー構造

### Layer 1 ブランド
外から見える名前。

人の意識を未来へ運ぶ入口。

```md
エネルギー循環OS
```

### Layer 2 本質
内側で実際にやっていること。

```md
接続OS
```

## エネルギー循環OSとの関係
[[02_PRINCIPLES/エネルギー循環OS|エネルギー循環OS]] は、外から見えるブランド名として強い。

理由:
- 人が得たい状態を表している
- 「循環」という結果を想像できる
- 接続という内部構造より、未来の状態に意識が向く

内部構造は、接続OSである。

```md
表の名前: エネルギー循環OS
内部エンジン: 接続OS
仕組み: 部品 → 接続 → 一致 → 循環 → エネルギー効率
```

## HOTEL JOYとの関係
「宿泊施設」は機能名。

「HOTEL JOY」は、使った後の状態を想像させる名前。

JOYという名前は、泊まる機能ではなく、気持ちの状態に接続する。

## Human Communication OSとの関係
[[02_PRINCIPLES/Human Communication OS|Human Communication OS]] では、言葉は相手の安心と接続を作る入口である。

同じ内容でも、最初の名前、呼び方、順番で相手の意識の向きが変わる。

悪い方向:
```md
クレーム
↓
責められている
↓
防御
```

良い方向:
```md
困っていること
↓
理解
↓
安心
↓
接続
```

## 投資家OSとの関係
「株スクリーニング」は機能名。

「投資家OS」は、会社を見る視点と未来の判断状態を表す名前。

人は、指標の一覧ではなく、会社の構造が見える状態に惹かれる。

## AI Agentとの関係
Agent名は、機能説明だけでなく、意識の向き先を作る。

例:
- Brain Curator Agent: 情報を増やすのではなく、整える意識へ接続する
- Compass Agent: 迷いではなく、現在地と次の1手へ接続する
- Identity Agent: 性格診断ではなく、本質とワクワク源へ接続する
- Communication Agent: 文章作成ではなく、理解と安心へ接続する

## 名前を決める時の問い
- この名前は、機能だけを説明していないか
- この名前は、使った後の状態を想像させるか
- この名前は、人の意識をどこへ運ぶか
- この名前は、接続したい未来へ向かっているか
- 内部構造名とブランド名を混同していないか
- ユーザーがエネルギーを感じる言葉か

## 避けたい名前
- 機能だけの名前
- 管理を強く感じる名前
- 直す、訓練、改善しろを連想させる名前
- 使った後の状態が見えない名前
- 英語だけで自分が直感的に分かりにくい名前

## 使いたい名前
- 未来の状態が見える名前
- 意識が前へ動く名前
- 接続、一致、循環を感じる名前
- 自分にも他人にも説明しやすい名前
- 日本語でも自然に理解できる名前

## AIへの指示
AIは、名前を決める時に機能だけで判断しない。

次の順番で考える。

```md
1. 使った後に得られる状態
2. 人の意識をどこへ接続したいか
3. 表に出すブランド名
4. 内部構造名
5. 実装上の名前
```

## 注意
名前は飾りではない。

名前は、人の意識を接続する入口である。

良い名前は、使う前から循環を始める。
