# Flow Bridge OS

## 位置づけ
Flow Bridge OSは、発見と現実で使う行動を接続するための操作OSである。

ブランド名は `Flow Bridge`。

内部構造名は `Execution Connection`。

## このファイルが必要な理由
今のMrBrainには、発見、思想、保存はかなり揃ってきた。

しかし、次の部分が詰まりやすい。

```md
発見
↓
思想
↓
保存
↓
？
↓
現実で使う
```

この `？` を接続するために、Flow Bridge OSを作る。

## 中核原則
発見は、保存して終わりではない。

現実で使える形に変換して、初めて循環が始まる。

Flow Bridge OSの役割は、次の1つ。

```md
発見と現実を接続する。
```

## 現在の上位方針
Flow Bridge OSは、[[07_SYSTEM/MrBrain Runtime Loop|MrBrain Runtime Loop]] の Sense / Route を補助する部品として扱う。

発見を現実へ流す前に、必要に応じて [[02_PRINCIPLES/Blueprint Engine|Blueprint Engine]] と [[07_AGENTS/Blueprint Reviewer|Blueprint Review]] へ接続する。

## 避けたい状態
- 発見して満足する
- 保存して止まる
- 調べるだけで終わる
- 情報が多すぎて使えない
- どのAIへ投げるか毎回迷う
- どこへ保存するか毎回迷う
- 実装前にまた思想へ戻り続ける

## Bridge Layerの5つの問い
Flow Bridgeは、毎回この5つだけを見る。

```md
1. 何を調べる？
2. 誰に聞く？
3. どのAIへ投げる？
4. どのファイルへ保存する？
5. どう使う？
```

## 入力
ユーザーは長く説明しなくてよい。

最小入力:
```md
これ教えて。
```

少し詳しい入力:
```md
これ、使えるようにしたい。
```

さらに詳しい入力:
```md
この発見を、MrBrainに保存して、次に何をすればいいかまで整理して。
```

## 自動判定カテゴリ
入力を受けたら、まず分類する。

| 種類 | 保存先 | 次の処理 |
|---|---|---|
| 思想 | `02_PRINCIPLES` | 原則化、判断基準化 |
| 知識 | `06_KNOWLEDGE` | 要約、出典、使い道 |
| 事業 | `03_BUSINESS` | 関連会社、原本場所、次の確認 |
| プロジェクト | `04_PROJECTS` | 目的、滞り、接続する部品、次の1手 |
| 実装設計 | `02_PRINCIPLES/Blueprint System.md` または対象プロジェクト | Blueprint化 |
| Agent | `07_AGENTS` | 役割、読むファイル、出力テンプレート |
| テンプレート | `07_SYSTEM/テンプレート集.md` または関連正本 | 再利用できる型にする |
| AI指示 | `07_SYSTEM` または関連Agent | プロンプト化 |
| タスク | `00_HOME/TASKS.md` | 接続タスク化 |
| 決定 | `07_SYSTEM/DECISION_LOG.md` | 理由、やめた案、リスクを記録 |

## どこから情報を集めるか
順番は固定する。

```md
1. MrBrain内の既存ファイル
2. ユーザーが貼った文章
3. 添付ファイル
4. Google Drive / スプレッドシート
5. NotebookLM用のソース
6. 公式サイトや一次情報
7. Web検索
```

注意:
- 税務、法律、医療、金融、最新情報は必ず一次情報や公式情報を優先する。
- 個人情報、税務情報、契約情報は原本を丸写ししない。
- URLだけで中身を見たことにしない。

## どのAIへ投げるか
| 目的 | 使う先 |
|---|---|
| 考えを深める | ChatGPT |
| MrBrainへ保存する | Codex |
| ファイルを作る、修正する | Codex |
| ソース資料を読ませる | NotebookLM |
| Google DriveやSheetsを確認する | Google Drive / Sheets接続 |
| 実装やコード修正 | Codex / Cursor |
| 長文資料の学習 | NotebookLM |

## Codexへ投げる文章テンプレート
```md
この内容をMrBrainへFlow Bridge OSに従って接続してください。

やってほしいこと:
1. 既存ファイルを確認
2. 内容を分類
3. 新規作成、既存追記、統合候補を判断
4. 保存先を決める
5. 次に現実で使う1手を作る
6. 必要ならAgent、テンプレート、プロンプトに変換

禁止:
- 推測で書き切らない
- 個人情報、税務情報、契約情報を雑に扱わない
- 移動、削除、リネームは勝手にしない

原文:
```

## NotebookLMへ渡す資料にする場合
NotebookLMは保存庫ではない。

NotebookLMへ渡すのは、学習や検索に使うソース。

残す正本はMrBrain。

Flow Bridgeでは、次を分ける。

```md
MrBrain: 正本、判断基準、決定ログ
NotebookLM: 読ませる資料、調査ソース
Codex: 保存、実装、検証
ChatGPT: 壁打ち、概念整理
```

## 出力形式
Flow Bridgeの出力は、必ず次の形にする。

```md
## 分類

## 今の滞り

## 接続する部品

## 情報を集める場所

## 保存先

## 使うAI

## Codexへの指示

## 再保存する場所
```

## 判断基準
- これは現実で使える形になったか
- Blueprintが必要な段階か
- 次の1手があるか
- 保存先が明確か
- どのAIへ投げるか決まっているか
- 調査ソースが明確か
- 再保存する場所があるか
- 思想で止まらず、操作へ接続できているか
