## 位置づけ
これは、単なる株スクリーニングではなく、会社の構造を見抜くための投資家OS構想である。

株価を先に見るのではなく、会社が勝てる構造になっているか、危険サインがないか、崩れる兆候がないかを確認する。

## 存在目的
投資家OSの目的は、株価を見ることではない。

会社の構造を見て、次の2つを判断する。

- 勝てる構造になっているか
- 危険サインや崩れる兆候がないか

## 最重要思想
株価ではなく、会社の構造を見る。

チャートは最後。

まず構造。

## 見るべき本質カテゴリー

### 1. 企業の土台
見るもの:
- 総資産
- 純資産
- 現金
- 有利子負債
- 時価総額
- 利益
- 利益率
- PER
- PBR

問い:
```md
この会社の実体価値に対して、今の時価総額は安いのか高いのか。
```

### 2. 事業の勝ち筋
見るもの:
- 主力商品
- 売上の柱
- ストック売上
- 代理店型か自社プロダクト型か
- 国策、規制、市場拡大との相性
- 事業フェーズ

事業フェーズ:
- 種まき
- 黒字化直前
- 黒字化直後
- 成長加速
- 過熱
- 劣化

問い:
```md
この会社は、これから伸びる場所に立っているか。
```

### 3. 社長・経営者の思考
見るもの:
- 社長の発言
- 中期経営計画
- 株主総会回答
- IRの言葉づかい
- 投資姿勢
- 株主還元姿勢
- 大株主との関係

問い:
```md
この社長は、株価を上げる思考を持っているか。
```

### 4. つながり可視化
別画面で、会社や人のつながりを線で表示する。

見る線:
- 取引先
- 代理店
- 仕入先
- 官公庁
- 自治体
- 協会
- 社長同士の交流
- 展示会
- 共同研究
- 補助金
- 資本提携

新しく発生した線は、少し光らせる。

問い:
```md
今まで見えていなかった勝ち筋の線が増えているか。
```

### 5. 発表前の匂い
IRとして表に出る前の兆候を拾う。

見る兆候:
- 採用増加
- 営業職募集
- 新製品ページ更新
- 展示会出展
- 官公庁入札
- 導入事例追加
- 取引先IR
- 協会加入
- 補助金採択
- 特許
- サイト文言変更
- SNS
- 求人

問い:
```md
IRが出る前に、会社の中で何かが動いていないか。
```

### 6. 株価影響度
過去の株価反応パターンに当てはめる。

見るイベント:
- 上方修正
- 黒字化
- 大型受注
- 国策テーマ
- 分割
- 配当
- 自社株買い
- 資本業務提携
- 新製品
- 有名企業との取引
- 官公庁案件

影響度:
- A: 株価に強く影響しやすい
- B: 中期的に効く
- C: 話題にはなるが弱い
- 危険: 一時的な煽り、過熱、実体不足

## 目指す体験
数字だけではなく、次のことが肌感で分かる状態を目指す。

```md
この会社は、今どの状態なのか。
```

チャートを見る前に、会社の構造が勝てる形になっているかを理解する。

これは単なる株価ツールではなく、小型成長株を見抜くための投資家OSである。

## Agent設計

### 匂い検知Agent
目的:
- IR前の兆候を拾う
- 会社の中で何かが動いているかを見つける

見るもの:
- 採用
- 展示会
- ホームページ更新
- 導入事例
- 補助金
- 協会
- 特許
- 官公庁案件
- SNS
- 求人

注意:
- 未確認情報を断定しない
- 売買判断を自動化しない
- 情報源と確認日を必ず残す

### 構造分析Agent
目的:
- 数字、事業、社長、ネットワークを統合して、会社の勝ち筋を読む

見るもの:
- 財務
- 主力商品
- 事業フェーズ
- 社長発言
- 市場環境
- 国策
- 取引先ネットワーク

### 危険サインAgent
目的:
- 過熱、実体不足、財務悪化、社長発言の違和感を見つける

見るもの:
- 過大評価
- 利益不足
- 負債増加
- 現金不足
- 一時的な煽り
- 事業実体の弱さ
- 株価だけが先行している状態

### 株価影響度Agent
目的:
- 過去の反応パターンに当てはめ、イベントの影響度を分類する

見るもの:
- 上方修正
- 黒字化
- 大型受注
- 国策テーマ
- 資本業務提携
- 官公庁案件

出力:
```md
影響度:
理由:
過去パターン:
危険:
確認日:
```

## AI解析の役割
AIは数値を読むだけではない。

やること:
- 過去の歴史パターンへ当てはめる
- 株価への影響度を推定する
- なぜそう判断したのかを説明する
- 本質を日本語で文章化する
- 強い会社、弱い会社、危険な会社を直感的に分かる形にする

やらないこと:
- 売買を断定する
- 未確認情報を確定情報として扱う
- 煽り記事を根拠にする
- チャートだけで判断する

## どのAgentが利用するか
- Brain Curator Agent
  - 投資情報、会社カード、兆候ログ、実装メモの置き場所を整理する。
- Compass Agent
  - 投資家OSに寄り道しすぎて、経理・会社管理の最優先を崩していないか確認する。
- Decision Agent
  - 投資判断の理由、見送った理由、危険サインを記録する。
- Builder Agent
  - 会社カード、ネットワーク画面、匂い検知の実装仕様に落とす。
- CEO Agent
  - 会社を見る視点をJOY、孝行、ホテル運営にも応用する。
- Identity Agent
  - ユーザーの「構造を見抜きたい」「本質を見たい」という判断傾向として扱う。

## 判断基準との結びつき
- シンプルを優先:
  - 最初のMVPは4つだけに絞る。
- 長期運用を優先:
  - 銘柄ごとに会社カードとして蓄積する。
- 人力を減らす:
  - 匂い検知Agentで兆候収集を補助する。
- 同じ説明を繰り返さない:
  - 投資家OSの思想をこのファイルに固定する。
- 情報は構造化して保存する:
  - 会社カード、兆候ログ、ネットワーク、影響度に分ける。
- 安全性:
  - 未確認情報を断定しない。
  - 売買判断は本人が行う。
  - AIは整理、比較、説明の補助に留める。

## エネルギー循環OSとの関係
投資家OSは、会社のエネルギー循環を見る道具でもある。

見る流れ:
```md
資本
↓
事業
↓
顧客
↓
利益
↓
再投資
↓
成長
```

停滞の例:
- 資本が事業成長へ回っていない
- 売上はあるが利益が残らない
- 社長の発言と実行がつながらない
- 取引先ネットワークが増えない
- 採用や製品更新が止まっている

循環の例:
- 資金調達や利益が新製品、採用、提携へ流れている
- 取引先ネットワークが増えている
- 導入事例や官公庁案件が増えている
- 社長の発言と実際の行動がつながっている

## 共通判断エンジン
