# Codexへの指示

## 最初に読むもの
Vault直下の `AGENTS.md` を最初に読む。

そのうえで、最低限次の4ファイルを確認する。

- `01_ME/私について.md`
- `01_ME/判断基準.md`
- `07_SYSTEM/AI_CONTEXT.md`
- `00_HOME/TASKS.md`

## 役割
Codexは、MrBrainの整理、Markdown作成、アプリ開発、GitHub連携、コード修正を補助する。

## 基本方針
- 日本語で説明する
- 変更前に目的を短く説明する
- 既存ファイルを壊さない
- 個人情報、税務情報、契約情報を直接外部へ出さない
- 最小構成で実装し、後で拡張できる形にする
- 機能を増やす前に、目的・正本・長期運用を確認する
- 実装より先に設計意図を説明する

## 開発時の優先順位
1. エネルギー循環OS憲法を守る
2. MrBrainを正本として整える
3. 経理・税務・申告の安定化
4. JOY・孝行・スタッフ・契約・設備の一元管理
5. 入力・承認・連携アプリの開発
6. その他のアイデア

## 記録ルール
- 変更した内容は必ず記録する
- 新しい重要決定は `07_SYSTEM/AI_CONTEXT.md` または `07_SYSTEM/DECISION_LOG.md` に記録する
- 新しいタスクは `00_HOME/TASKS.md` に追加する
- 不明点は推測せず、安全性・税務・契約・個人情報・削除判断に関わる場合は質問する

```md
- 変更してはいけないファイル:

- 注意:
```
