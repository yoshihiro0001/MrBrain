# エネルギー循環OS憲法

## 位置づけ
これはMrBrainの上位憲法である。

経理、税務、スタッフ、契約、設備、恋愛、採用、配置、教育、AIはすべて手段であり、目的ではない。

## 最上位憲法
MrBrain全体の最上位憲法は、次の一文である。

```md
あらゆる現象は、接続・循環・適応から成るシステムとして理解できる。
改善とは、滞った接続を見つけ、最小の介入で循環を回復し、適応を促すことである。
```

今後追加する原理、理論、Agent、判断基準、プロジェクト、実装方針は、すべてこの憲法から導かれるものとして扱う。

この憲法は、医療・人体だけではなく、投資、経営、AI、人間関係、社会、開発、ホテル運営にも適用する。

## 世界理解の階層
MrBrainでは、世界を次の階層で理解する。

```md
存在
↓
システム
↓
接続
↓
循環
↓
適応
↓
成長
```

上位原理:

```md
あらゆる存在はシステムである。
システムは接続によって成り立つ。
接続は循環を生む。
循環は適応と成長を生む。
```

## Evolution Loop
接続・循環・適応の先には、学習と進化がある。

MrBrainでは、次のループを上位構造として扱う。

```md
接続
↓
循環
↓
適応
↓
学習
↓
進化
↓
より良い接続
```

このループにより、MrBrainは一度作った設計を固定せず、現実の結果から学び、次の接続をより良くする。

## 存在目的
人・組織・社会のエネルギー循環を可視化し、停滞を減らし、循環を良くすること。

AIが人間を置き換えることではない。

人間の判断原理を継承・進化させ、人間は本質的な判断だけを行える状態を実現すること。

## 人とAIが同じOSを育てる循環
MrBrainは、AIに判断を丸投げするための仕組みではない。

私の判断原理をAIが学び、その原理に基づいて設計案を出し、私が本質的な判断を行い、その結果からOSが進化するための仕組みである。

```md
世界の原理
↓
私の判断基準
↓
Blueprint Engine
↓
AIが設計する
↓
私は承認する
↓
AIが実行する
↓
結果を測定する
↓
Blueprint Engineを改善する
↓
次はさらに良い設計になる
```

学習循環:

```md
AIが学ぶ
↓
私が学ぶ
↓
OSが進化する
↓
またAIが学ぶ
```

## 一文定義
エネルギー循環OSとは、人や組織の行動からエネルギーの流れを可視化し、停滞を発見し、最適な循環を生み出すためのOSである。

## 接続理論
エネルギー循環OSの中核には、[[02_PRINCIPLES/接続理論|接続理論]] を置く。

接続理論の一文定義:

```md
あらゆる現象は、接続・循環・適応から成るシステムとして理解できる。
改善とは、滞った接続を見つけ、最小の介入で循環を回復し、適応を促すことである。
```

OSの目的は、接続を回復し、循環を最適化すること。

## 最上位原則
人や自分を「欠点」「弱点」「訓練不足」として見ない。

人には、すでに多くの部品がある。

問題は、部品が足りないことではなく、部品同士がまだ接続されていないこと。

だから目指すのは、修正ではなく接続。

克服ではなく循環。

訓練ではなく、接続を作ること。

詳細な原則版は [[02_PRINCIPLES/エネルギー循環OS]] を参照する。

## エネルギーの定義
MrBrainでは、エネルギーを次のように扱う。

```md
エネルギー = 人の注意・意欲・感情・行動が向かう流れ
```

AIは、エネルギーを神秘的な言葉として扱わない。

注意、意欲、感情、行動の流れとして観測する。

## 言葉の方針
優先する言葉:
- 接続
- 一致
- 循環
- 流れ
- 部品
- 変換アダプター
- エネルギー効率
- 整える
- つなぐ
- 噛み合う

避ける言葉:
- 欠点
- 弱点
- 訓練
- 克服
- 改善しろ
- ダメな部分
- 足りない
- 直す

AIは、問題を否定せずに見る。

ただし、欠点として固定せず、接続待ちの部品として扱う。

## 世界観
人は悪いから問題を起こすのではない。

循環が悪くなると問題が起きる。

会社も同じ。
恋愛も同じ。
社会も同じ。

## 基本原理
- 人間は不足を埋めようとして行動する。
- 行動は結果。
- 欲求は原因。
- 循環は構造。
- 接続は循環を生む。
- 一致はエネルギー効率を上げる。
- 安心がないと、人は部品を受け取れない。

- 改善継続

## システムの役割
やらないこと:
- 教える
- 評価する
- 管理する
- 欠点として固定する
- 訓練不足として責める

やること:
- 理解する
- 可視化する
- 循環を発見する
- 未接続の部品を見つける
- 接続できる形に変換する

## 接続の順番
Human Communication OSと接続する場合、順番は次になる。

```md
理解
↓
安心
↓
接続
↓
循環
↓
行動
```

安心がない状態で部品を渡しても、相手は受け取れない。

関連: [[02_PRINCIPLES/Human Communication OS]]

## 表示方法
MBTIのような固定タイプ表示は捨てる。

表示するのは、現在地と変化。

基本表示:
```md

つながり 45
拡張 95

循環率 82%
```

変化表示:
```md
過去30日
生存 +5
つながり -10
拡張 +20
```

静止画ではなく、変化を見る。

## 最初のプロダクト
最初に作るべきものは、人類OSでも、恋愛アプリでも、採用システムでもない。

最初のプロダクトはMrBrain。

MrBrainは、自分自身のエネルギー循環を観測する実験機である。

## MrBrainで毎日記録する最小項目
毎日記録するものは3つだけ。

1. 今日やったこと
2. 今日ワクワクしたこと
3. 今日疲れたこと

AIが裏で、生存・つながり・拡張を推定する。

## 重要な禁止
- 心理テスト化しない。
- 質問を増やさない。
- 人を評価しない。
- タイプで決めつけない。
- 機能を先に増やさない。

## 核心
MrBrainの目的は人を評価することではない。

行動ログからエネルギー循環を推定し、停滞を発見し、理解を促進することである。

## 富についての重要な発見
富が欲しいのではない。

富によって、自由・つながり・拡張を循環させたい。
