# MrBrainスキル化構想

## 目的
CodexやAIに毎回同じ説明をしなくても、MrBrainの思想・構造・安全ルール・記録形式を理解して動けるようにする。

将来的には、MrBrain専用のスキルを作る。

## なぜスキル化するか
- 同じ説明を何度もしたくない
- 経理、税務、契約、スタッフ、設備、健康、思考ログが混ざるため、安全ルールが必要
- MrBrainには独自の憲法がある
- エネルギー循環OSの思想をAIに毎回守らせたい
- ファイルの置き場所をAIが迷わないようにしたい

## スキルに入れるべき内容
### 1. MrBrainの目的
- 自分の人生、会社、家、契約、経理、スタッフ、健康、開発、思考を一元管理する
- AIへ同じ説明を繰り返さない
- 正本はMrBrain、業務データはスプレッドシート、操作はアプリと分ける

### 2. エネルギー循環OS憲法
- 目的は、人・組織・社会のエネルギー循環を可視化すること
- 評価ではなく理解
- 質問ではなく行動ログ
- 生存・つながり・拡張の3つだけ

### 3. 安全ルール
- パスワードを書かない
- 税務・契約・給与情報は原本ではなく索引
- 個人情報は最小限
- 未確認の税務判断を断定しない

### 4. ファイル配置ルール
- 憲法・AIルール: `07_SYSTEM`
- 自分の傾向・日次ログ: `01_ME`
- 事業: `03_BUSINESS`
- プロジェクト: `04_PROJECTS`
- 知識: `06_KNOWLEDGE`
- 古いもの: `99_ARCHIVE`

### 5. 日次ログ処理
入力:
- 今日やったこと
- 今日ワクワクしたこと
- 今日疲れたこと

出力:
- 生存
- つながり
- 拡張
- 循環率
- 根拠になった行動ログ
- 停滞サイン

### 6. Human Communication OS
- 正本: `02_PRINCIPLES/Human Communication OS.md`
- 目的: 返信、依頼、接客、クレーム対応、スタッフ指導、AI指示を、理解と安心を先に置く形へ整える
- 中核原則: 人は問題解決の前に理解されたい
- 将来スキル化する場合は、長文を「本音1行」「相手の感情仮説」「返信案」「注意点」へ圧縮する

### 7. Flow Bridge OS
- 正本: `02_PRINCIPLES/Flow Bridge OS.md`
- Agent: `07_AGENTS/Flow Bridge Agent.md`
- 目的: 発見、保存、調査、AI指示、実装、現実で試す1手を接続する
- 中核原則: 発見は、現実で使える形に変換して初めて循環する
- 将来スキル化する場合は、「これ教えて」だけで、分類、保存先、使うAI、Codex指示、現実で試す1手まで出す

### 8. Blueprint System
- 正本: `02_PRINCIPLES/Blueprint System.md`
- Agent: `07_AGENTS/Blueprint Agent.md`
- 目的: 目的、現在地、完成形、必要部品、実装手順、テスト方法、保存場所を1枚の設計図にする
- 中核原則: 思想の次に必ずBlueprintを通す
- 将来スキル化する場合は、経理自動化、Booking.com効率化、アプリ開発、スキル化を同じ設計図で始められるようにする

## 最初に作るなら
まずは大きな自動化ではなく、MrBrain用の軽いスキルから始める。

### Skill案
```md
name: mrbrain
description: MrBrain Vaultの整理、エネルギー循環ログ、経理・事業・開発の索引化、安全ルールに沿った記録を支援する。
```

### 最初の機能
- 今日の会話から重要発見をMrBrainに保存する
- 日次エネルギー循環ログを作る
- 新しい情報の置き場所を判断する
- AI_CONTEXTを更新する
- 個人情報や税務情報を安全な形に要約する
- Human Communication OSに沿って返信案を作る
- Flow Bridge OSに沿って、発見を現実で使う次の1手へ接続する
- Blueprint Systemに沿って、実装前の設計図を作る

## まだやらないこと
- 自動で全ファイルを動かす
- 勝手に外部送信する
- 税務判断を確定する
- 人を評価するスコアリングにする

## 最適な順番
1. まずMrBrain内で手動運用する
2. 日次ログが10日分たまる
3. よく使う処理をテンプレート化する
4. Codexスキルにする
5. その後、JOYやスタッフ管理へ広げる

## 返信スキル候補
```md
name: human-communication-os
description: 長い文章や違和感を読み、Human Communication OSに沿って、接客・依頼・クレーム対応・スタッフ指導・AI指示の返信案を作る。
```

## Flow Bridgeスキル候補
```md
name: flow-bridge
description: ユーザーの発見や相談を、分類、保存先、使うAI、Codex指示、現実で試す次の1手まで接続する。
```

## Blueprintスキル候補
```md
name: blueprint-builder
description: ユーザーの目的や相談を、目的、現在地、完成形、必要部品、実装手順、テスト方法、保存場所を含む設計図に変換する。
```

## 判断
スキル化は有効。

ただし、今すぐ高度なスキルを作るより、まずMrBrain内の憲法とログ形式を固定する。
その後、繰り返し作業が見えた時点でスキル化するのが一番安全でコスパが良い。
