# Codex便利な使い方

## 目的
Codexを、MrBrain・Obsidian・GitHub・開発の実務補助として使いこなす。

## Codexの得意なこと
- ファイルを読む
- Markdownを整理する
- コードを修正する
- ターミナルで確認する
- GitHub連携の準備をする
- スプレッドシートや資料の構造を分析する
- 変更内容を記録する

## MrBrainでの基本運用
Codexには、まず `AGENTS.md` を読ませる。

その後、作業に必要な関連ファイルを読ませる。

```md
AGENTS.mdを読んでから、以下をお願いします。
目的:
対象ファイル:
やってほしいこと:
やってほしくないこと:
完了条件:
```

## 便利な依頼例
### 1. 現状分析だけ頼む
```md
改善案はまだ不要です。
まず現状分析だけしてください。
重複、不足、正本候補を整理してください。
```

### 2. MrBrainに保存させる
```md
この会話で重要な発見を、MrBrainの適切な場所に保存してください。
必要ならHOME、AI_CONTEXT、TASKSにもリンクしてください。
```

### 3. GitHub保存の前に安全確認させる
```md
GitHubへ保存する前に、個人情報・税務情報・契約情報・秘密情報が含まれていないか確認してください。
```

### 4. 開発前に設計意図を出させる
```md
いきなり実装せず、まず設計意図と最小構成を説明してください。
その後、必要なファイルだけ変更してください。
```

### 5. 変更後に記録させる
```md
変更内容をDECISION_LOGまたはTASKSに記録してください。
```

## Codexに向いている作業順
1. 現状確認
2. 目的整理
3. 最小構成の提案
4. ファイル編集
5. 検証
6. 記録
7. 次の1手

## やらせすぎない方がいいこと
- 税務判断の確定
- 契約内容の法的判断
- 個人情報の全文保存
- いきなり大規模リファクタ
- 目的が曖昧な新機能追加

## MrBrainでの合言葉
```md
機能より先に憲法。
実装より先に設計意図。
保存より先に安全確認。
```

## 自分向けの使い方
あなたは、同じ説明を繰り返すのが苦手。

だからCodexには、毎回ゼロから説明するのではなく、次の順で渡す。

1. `AGENTS.md`
2. `07_SYSTEM/AI_CONTEXT.md`
3. 関連するプロジェクトファイル
4. 今回の依頼

これで、説明量を減らしつつ精度を上げる。
